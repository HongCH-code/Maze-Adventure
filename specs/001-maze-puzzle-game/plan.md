# Implementation Plan: Maze Puzzle Game

**Branch**: `001-maze-puzzle-game` | **Date**: 2026-03-06 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-maze-puzzle-game/spec.md`

## Summary

Build a 2D maze puzzle game with 20 progressively difficult levels targeting children age 6-7. Players navigate mazes, solve first-grade math puzzles at locked gates, and collect items. The game runs on desktop and mobile browsers as a PWA, built with Phaser 3 + TypeScript + Vite. Level definitions are data-driven (JSON), and the item/puzzle systems use registry patterns for extensibility.

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode)
**Primary Dependencies**: Phaser 3 (game engine), Vite (build tool), vite-plugin-pwa (PWA support)
**Storage**: Browser localStorage for player progress and scores
**Testing**: Vitest for unit tests on game logic (maze generation, puzzle validation, progress tracking)
**Target Platform**: Modern desktop browsers (Chrome, Firefox, Safari, Edge), mobile browsers (iOS Safari 15+, Android Chrome)
**Project Type**: Web game (single-page application)
**Performance Goals**: 60fps rendering, <5s initial load on 4G, <100ms input response
**Constraints**: Offline-capable via service worker, all assets cached, no server dependency
**Scale/Scope**: 20 levels, ~5 entity types, ~4 scene types, single-player only

### Source Code (repository root)

```text
src/
├── main.ts              # Phaser game config and bootstrap
├── scenes/
│   ├── BootScene.ts     # Asset preloading
│   ├── TitleScene.ts    # Title/start screen
│   ├── LevelSelectScene.ts  # Level selection map
│   ├── GameScene.ts     # Main gameplay (maze rendering, player movement, items)
│   ├── PuzzleScene.ts   # Math puzzle overlay (launched over GameScene)
│   └── LevelCompleteScene.ts  # Score summary and next-level prompt
├── systems/
│   ├── MazeGenerator.ts # Maze generation algorithms (DFS, recursive backtracker)
│   ├── PuzzleManager.ts # Puzzle type registry and question generation
│   ├── ItemRegistry.ts  # Item type definitions and collection logic
│   ├── InputManager.ts  # Unified keyboard + touch input handling
│   ├── ProgressManager.ts  # localStorage read/write for player progress
│   └── ScaleManager.ts  # Responsive scaling configuration
├── data/
│   ├── levels/          # JSON level definitions (level-01.json ... level-20.json)
│   └── puzzles/         # Puzzle pool definitions per difficulty tier
├── assets/
│   ├── sprites/         # Player, walls, items, gates, UI elements
│   ├── tilemaps/        # Tileset definitions
│   └── audio/           # Sound effects (collect, correct, incorrect, complete)
└── types/
    └── index.ts         # Shared TypeScript interfaces and types

public/
├── index.html
├── manifest.json        # PWA manifest
└── sw.js                # Service worker (generated by vite-plugin-pwa)

tests/
├── MazeGenerator.test.ts
├── PuzzleManager.test.ts
├── ItemRegistry.test.ts
└── ProgressManager.test.ts
```

**Structure Decision**: Single project structure. This is a client-only game with no backend. All source code lives under `src/`, with clear separation between scenes (UI/rendering), systems (game logic), data (level/puzzle definitions), and assets (media files).
